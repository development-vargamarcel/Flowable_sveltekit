# Examples

## Example: post-change documentation updates
1. Implement a code or configuration change.
2. Increment version from `00.00.02` to the next semantic value.
3. Update `usage.md` to describe the updated behavior.
4. Add an example in this file.
5. Update pseudocode in `LogicExplainedWithPseudocode.md`.
